# Telegram-бот «Робо-питомцы»: тренажёр таблицы умножения и деления для Александры

**Дата:** 2026-07-05
**Статус:** утверждено

## Цель

Ребёнок 10 лет (Александра) тренирует таблицу умножения и деления 2–9 три раза в день
по 5–10 примеров за сессию. Бот делает это в игровой форме, мотивирует, помогает
запоминать через подсказки-приёмы и интервальное повторение, награждает звёздами и
коллекционными карточками робо-питомцев (стиль: «Дроны-убийцы» × милые животные).

## Ключевые решения

- **Платформа:** Telegram-бот. У Александры свой аккаунт; маме (Елене) приходят отчёты.
- **Хостинг:** GitHub Actions, **публичный** репозиторий (минуты бесплатны без лимита).
- **Расписание:** окна тренировок в 10:00, 14:00, 17:00 по Кипру (07:00, 11:00, 14:00 UTC).
- **Механика:** живой диалог в окне (long-polling до 25 минут). Ответы Александра
  **набирает цифрами** — активное вспоминание, а не выбор из вариантов.
- **Охват:** вся таблица 2–9, умножение и деление, адаптивно (интервальное повторение).

## Игровая механика сессии

- Сессия = «вылет робо-питомца на задание»: 10 примеров, минимум 5
  (после 5-го можно завершить командой «стоп» — засчитывается).
- Правильный ответ: ⚡ + мотивационная фраза с именем («Мощно, Александра! 🔥»).
- Серия из 3+ правильных подряд — комбо-бонус (доп. очки, особые фразы).
- Ошибка: без ругани. Бот даёт **подсказку-приём** и вторую попытку;
  пример возвращается ещё раз в конце сессии.
- Если Александра пишет боту вне окна — бот молчит до следующего окна;
  в каждом приветствии указано время следующей тренировки.

### Подсказки-приёмы (примеры)

- 7×8: «5, 6, 7, 8 → 56 = 7×8»
- ×9: приём с пальцами (загни N-й палец: слева десятки, справа единицы)
- ×4: «удвой и ещё раз удвой»
- ×5: «половина от ×10»
- Деление: «сколько раз по B помещается в A? Вспомни B×? = A»

## Запоминание: система Лейтнера

- Каждый факт (a×b для 2≤a,b≤9 и соответствующее деление) имеет уровень 0–4.
- Ошибка → уровень вниз (факт приходит чаще). Быстрый верный ответ → уровень вверх.
- Выбор примеров для сессии: в первую очередь факты с низким уровнем и те,
  что давно не повторялись; немного новых/случайных для разнообразия.
- Состояние хранится в `progress.json` в репозитории, коммитится после каждой сессии
  (тот же паттерн, что `seen.json` у dota2-news-bot).

## Награды

- **Звёзды за сессию:** до 3 (по точности: ≥90% — 3⭐, ≥70% — 2⭐, иначе 1⭐ за участие).
- **Итог дня** (подводится в вечернем окне 17:00 после сессии): при 2+ сессиях
  и 6+ звёздах открывается новая **карточка робо-питомца**.
- **Коллекция:** ~30 карточек (кибер-котята, дроны-щенята, робо-лисы…) с редкостью
  (обычные/редкие/легендарные). Картинки генерируются заранее через Gemini
  и лежат в репо (`cards/`).
- **Streak:** серия дней с выполненной нормой; особые карточки за 3, 7, 14, 30 дней.
- Команда `/коллекция` (и `/collection`) — показать собранные карточки и звёзды.

## Отчёт для мамы

Раз в день (в вечернем запуске) бот отправляет Елене краткий отчёт:
количество сессий, точность, текущий streak, топ-3 «западающих» факта.

## Техническая архитектура

- **Стек:** TypeScript + Node.js (как dota2-news-bot), без фреймворков —
  прямые вызовы Telegram Bot API (sendMessage, getUpdates, sendPhoto).
- **Запуск:** GitHub Actions workflow, cron 07:00/11:00/14:00 UTC + workflow_dispatch
  для ручного теста.
- **Цикл окна:** отправить приветствие → long-polling getUpdates до 25 минут →
  провести сессию → начислить награды → закоммитить `progress.json` → (вечером)
  итог дня + карточка + отчёт маме.
- **Секреты:** `TELEGRAM_BOT_TOKEN`, `CHILD_CHAT_ID`, `PARENT_CHAT_ID` — в GitHub Secrets.
- **Модули:**
  - `src/telegram.ts` — тонкий клиент Telegram API
  - `src/facts.ts` — генерация фактов, подсказки-приёмы
  - `src/leitner.ts` — интервальное повторение, выбор примеров
  - `src/session.ts` — конечный автомат сессии (приветствие → примеры → итог)
  - `src/rewards.ts` — звёзды, карточки, streak
  - `src/report.ts` — отчёт родителю
  - `src/state.ts` — чтение/запись `progress.json`
  - `src/index.ts` — точка входа (определяет окно: утро/день/вечер)

## Обработка ошибок и краевые случаи

- Cron GitHub Actions может запаздывать на 5–15 минут — это ок, приветствие придёт позже.
- Александра не ответила за 25 минут → сессия «пропущена», без негатива;
  в следующем приветствии — мягкая мотивация вернуться.
- Ответ не число («не знаю», стикер) → дружелюбная просьба написать цифрами + подсказка.
- Два запуска не пересекаются: окна разнесены на 3+ часа, каждый job ≤ 30 минут.
- Конфликт коммита `progress.json` — pull --rebase перед push.

## Тестирование

- Unit-тесты (vitest): leitner (повышение/понижение уровней, выбор примеров),
  facts (корректность примеров и подсказок), rewards (звёзды, условия карточек, streak).
- Интеграционно: мок Telegram API, прогон полной сессии конечного автомата.
- Ручной тест: workflow_dispatch с тестовым chat_id (мамин) до подключения Александры.

## Вне рамок (YAGNI)

- Веб-интерфейс, база данных, несколько детей, другие операции (сложение и т.п.),
  мгновенные ответы вне окон. При желании позже можно мигрировать на постоянный хостинг.
