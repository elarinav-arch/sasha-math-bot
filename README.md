# Робо-питомцы 🤖🐾 — тренажёр таблицы умножения для Александры

Telegram-бот, который 3 раза в день (примерно с 14:00, 17:00 и 19:00 по Кипру) зовёт
Александру потренировать таблицу умножения и деления: 5–10 примеров, подсказки-приёмы,
интервальное повторение, звёзды и коллекция карточек робо-питомцев.

## Настройка

1. **Бот:** напишите [@BotFather](https://t.me/BotFather) → `/newbot` → получите `TELEGRAM_BOT_TOKEN`.
2. **Chat ID:** Александра (со своего аккаунта) отправляет боту `/start`.
   Затем откройте `https://api.telegram.org/bot<TOKEN>/getUpdates` — в ответе будет
   `message.chat.id`. Это `CHILD_CHAT_ID`. Свой `PARENT_CHAT_ID` узнайте так же
   (напишите боту со своего аккаунта) — на него приходят вечерние отчёты.
3. **GitHub:** запушьте репозиторий (публичный), в `Settings → Secrets and variables → Actions`
   добавьте `TELEGRAM_BOT_TOKEN`, `CHILD_CHAT_ID`, `PARENT_CHAT_ID`.
4. **Проверка:** вкладка Actions → workflow `training` → `Run workflow` (slot = `morning`).
   Бот должен написать приветствие в течение минуты.

## Как это работает

- Три тренировки в день: около 14:00, 17:00 и 19:00 по Кипру. Внутри окна бот ждёт
  ответов до часа (long polling). Вне окон бот не отвечает.
- **Точное время — через внешний планировщик, не через GitHub cron.** GitHub Actions
  `schedule` — best-effort и на практике срабатывает нерегулярно (раз в 1–3 часа,
  в произвольную минуту), поэтому полагаться на него для конкретного времени нельзя
  (проверено: тренировки уезжали на час-два позже расписания). Реальный триггер —
  бесплатный [cron-job.org](https://cron-job.org/), который в 14:00/17:00/19:00 по
  Кипру дёргает GitHub REST API (`workflow_dispatch` с явным `slot`), запуская
  тренировку принудительно и точно вовремя. Настройка:
  1. Создать fine-grained PAT: `github.com/settings/personal-access-tokens/new` →
     Repository access → Only select repositories → этот репозиторий → Permissions →
     `Actions: Read and write`, `Contents: Read-only` → Generate token.
  2. На cron-job.org создать 3 задания (Method: `POST`, ежедневно, часовой пояс
     `Europe/Athens`/`Asia/Nicosia`, времена 14:00/17:00/19:00):
     - URL: `https://api.github.com/repos/elarinav-arch/sasha-math-bot/actions/workflows/train.yml/dispatches`
     - Headers: `Authorization: Bearer <PAT>`, `Accept: application/vnd.github+json`,
       `Content-Type: application/json`
     - Body: `{"ref":"main","inputs":{"slot":"morning"}}` (для второго и третьего
       заданий — `"midday"` и `"evening"` соответственно)
  - Внутренний GitHub cron (`*/10 11-19 * * *`, сплошные окна 14:00–17:00/17:00–19:00/
    19:00–22:00 по Кипру, см. `activeSlot` в `src/index.ts`) остаётся как подстраховка
    на случай, если внешний планировщик недоступен — если так, тренировка всё равно
    случится в течение дня, просто не строго по расписанию. Дедуп по дню
    (`progress.json`, поле `attemptedSlots`) не даёт провести один и тот же слот дважды.
  - Ручной запуск (`workflow_dispatch` со значением `slot` — так же, как это делает
    cron-job.org) форсирует конкретный слот, игнорируя и текущее время, и дедуп —
    удобно для тестирования.
- Прогресс хранится в `progress.json` и коммитится после каждой сессии.
- Дневная цель: 2+ сессии и 6+ звёзд → карточка робо-питомца. Серии дней дают
  особые карточки (3, 7, 14, 30 дней).
- **Бонусный раунд.** Если к вечернему слоту дневная цель не набрана, бот сам
  предлагает короткий раунд из 5 примеров с отдельным 20-минутным окном ожидания
  (`extendDeadline` в `TelegramIO`). Если Александра в нём поучаствует (ответила
  хоть на один пример) — карточка дня засчитывается в любом случае, независимо
  от результата раунда: это подстраховка от неудачного дня, а не ещё одно испытание.
- **Редкость карточек.** 55% обычные / 30% редкие / 15% легендарные — доля именно
  по УРОВНЮ редкости, а не по отдельной карте (`pickNewCard` в `src/rewards.ts`:
  сначала выбирается уровень, потом конкретная карта внутри него), поэтому реальная
  вероятность легендарки не зависит от того, сколько легендарных карт уже собрано.
- Команды в чате: ответ цифрами, `стоп` (после 5 примеров), `/коллекция`.

## Разработка

- `npm test` — тесты (vitest), `npm run typecheck` — проверка типов.
- Ручной запуск локально: `TELEGRAM_BOT_TOKEN=... CHILD_CHAT_ID=... SESSION_SLOT=morning npm start`.
